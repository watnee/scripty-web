Scripty API — link relations

Responses from /api use HAL. Standard IANA relations such as self keep their usual meaning; the project-specific relations below are namespaced scripty:* via a HAL curie, and each links here by its name. A client should follow these relations rather than hard-coding URLs — presence of a mutation relation also signals that the current user is allowed to perform it. Send Accept: application/prs.hal-forms+json to additionally receive _templates describing each mutation's HTTP method and fields.

Navigation

RelationLeads to
projectsThe collection of projects.
projectThe owning project of the current resource.
blocksThe screenplay blocks of a project.
charactersThe characters (people) in a project.
actorsThe actors, optionally scoped to a project.
actorThe actor cast in a character.
usersThe collection of users.
teamsThe collection of teams.
documentsA project's text documents (songs and notes).
documentA single text document.
songsA project's song documents.
songThe song a block or version belongs to.
songBlocksThe lyric blocks of a song.
notesA project's note documents.
commentCountsHow many comments sit on each element of a project, keyed by block id, as {"counts": {"12": 3}}. Advertised on a non-empty block collection so a client can mark the discussed elements without a request per element; elements with no comments are absent rather than zero.
songEditionsA song's named editions — an alternate lyric, a rewrite, a cut for a different scene. The song counterpart of editions, hung off the document rather than the project; the resource is shaped identically.
songEditionA single song edition.
versionsThe version history of a project or song.
songVersionsThe version history of a song, as the collection relation embedding its snapshots. A song snapshot reports the title and lyric line count where a screenplay's reports scenes and elements.
versionA single saved version.
headshotAn actor's headshot image. Present only when the actor has one.
forgotPasswordAsk for a password-reset email. POST {"email"}. Always answers 202 with the same message, whether or not the address is registered. Advertised on the 401 challenge, since nothing behind the sign-in can advertise it.
resetPasswordSet a new password using the token from a recovery email. POST {"token","password"}. GET the same URL with ?token= to ask whether a token is still good and whose account it belongs to.
setHeadshotStore a headshot for this actor. POST a multipart/form-data body with a headshot part holding a JPG, PNG, WebP or GIF of 5 MB or less. Answers with the refreshed actor.
removeHeadshotDELETE this actor's headshot. Present only when the actor has one. Answers with the refreshed actor.
syncStatusWhether a project changed since a given revision.
undoRedoStatusWhether undo/redo is currently available.
capitalizationPreferencesThe signed-in user's auto-capitalization settings.
contactSuggestionsName/email suggestions for a project's invite forms.
accountThe signed-in user's own account. Not an admin's view of someone else's.
passkeysThe passkeys registered to the signed-in user. Present only where passkeys are configured; registration itself is a browser-side WebAuthn ceremony, so the API lists and revokes only.
passkeyA single registered passkey.
editionsA project's named editions of the screenplay — a shooting draft, a table read, a production revision. editionId is accepted on blocks, imports and version history; this is how a client learns which ids exist.
editionA single script edition.
commentsThe comments left on one element.
commentA single comment.
activityA project's activity feed — who changed what, most recent first.
activityEntryA single entry in the activity feed.
invitationsThe outstanding invitations to a project.
invitationA single invitation.
trashThe trash a deleted thing went to. Advertised alongside delete so a client can say where the deletion can be undone from.
trashedProjectsProjects the signed-in user has deleted and can still restore.
trashedProjectA single deleted project.
deletedBlocksElements deleted from a project, still restorable.
deletedBlockA single deleted element.
deletedDocumentsSongs and notes deleted from a project, still restorable.
deletedDocumentA single deleted song or note.
deletedSongBlocksLyric lines deleted from a song, still restorable. Kept apart from deletedBlocks because a line and a screenplay element are restored by different services and carry different fields.
deletedSongBlockA single deleted lyric line. purgeAt is absent where the deployment keeps trashed lines until someone deletes them for good.
accessEveryone who can already see a project, with why (their role, or the team carrying it) and whether they can write. Not the same as invitations: role and team membership grant access without one.
accessUserOne person who can see the project.

Present only when the current user may perform them.

setDefaultPOSTMake this edition the one opened when no edition is named.
setPublishedPOSTMake this edition the one view-only readers see. Independent of the default, so a writer can work in a draft while readers stay on the last published cut.
purgeDELETEDelete one item from the trash for good. Not recoverable.
emptyTrashDELETEDelete everything in the trash for good. Not recoverable.
bulkSetTypePOSTRetype every element in a selection. Advertised on the block collection rather than a block, since it acts on a set of them.
bulkAddTagsPOSTAdd tags to every element in a selection.
bulkFormatPOSTSet alignment, font, style or highlight across a selection in one undo checkpoint. Each field applies only when present; style is a per-element toggle rather than a set, so a mixed selection comes back inverted. Pass clearHighlight to remove a tint, since a blank highlight means "leave alone".
bulkReplacePOSTFind and replace across a selection. find and replace are both literal — never regular expressions. Character cues are skipped unless includeCharacterCues is set, because a cue mirrors its person record.
bulkDeletePOSTDelete every item in a selection — elements when it rides on the block collection, songs when it rides on the document collection. Recoverable from the project's trash either way.
